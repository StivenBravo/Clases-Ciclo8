EXPLICACIÓN PASO A PASO - CARTA DE RESTAURANTE

Este proyecto simula una aplicación web para la carta de un restaurante, donde se pueden ver menús, añadir nuevos platos,
 tomar pedidos y guardar una factura.

Está dividido en una parte "cliente" (lo que ve el usuario en el navegador) y una parte "servidor" (pequeños servicios que manejan los datos).

---
1. CÓMO EJECUTAR EL PROYECTO
---

Para que todo funcione, necesitas tener Node.js instalado en tu computadora.

1.1. Abre tres terminales o consolas separadas en la carpeta de tu proyecto.

1.2. En la primera terminal, ejecuta el servidor de comidas:
     node server1.js
     (Verás el mensaje: Server 1 (Comidas) corriendo en http://localhost:3001)

1.3. En la segunda terminal, ejecuta el servidor de facturas:
     node server2.js
     (Verás el mensaje: Server 2 (Facturas) corriendo en http://localhost:3002)

1.4. En la tercera terminal, ejecuta el servidor de tragos:
     node server3.js
     (Verás el mensaje: Server 3 (Tragos) corriendo en http://localhost:3003)

1.5. Finalmente, abre el archivo `index.html` en tu navegador web (como Chrome, Firefox, etc.). Puedes hacer doble clic sobre él.

---
2. ARCHIVOS DEL CLIENTE (Lo que se ejecuta en el navegador)
---

*   `index.html`: Es el esqueleto de la página.
    - Usa Bootstrap 5 (BS5) para tener un diseño moderno y responsivo sin mucho esfuerzo.
    - Define las áreas principales: un formulario para añadir platos, dos secciones para los menús (comidas y tragos) y una columna para "Mi Pedido".
    - Carga los archivos CSS y JavaScript necesarios.

*   `css/style.css`: Contiene estilos personalizados muy básicos para mejorar la apariencia de las imágenes de las tarjetas, si las hubiera.

*   `js/main.js`: Es el motor de la página.
    - **Al cargar la página**: Usa `fetch` para pedir la lista de comidas a `server1.js` y la lista de tragos a `server3.js`.
    - **Renderizado**: Una vez que recibe los datos, crea dinámicamente las tarjetas (cards) de cada plato y trago y las muestra en la página.
    - **Añadir Plato Nuevo**: Escucha el evento 'submit' del formulario. Cuando se envía, toma los datos, los empaqueta en formato JSON y los envía con un `fetch` de tipo `POST` a `server1.js`. Si tiene éxito, vuelve a cargar el menú de comidas para que se vea el nuevo plato.
    - **Añadir al Pedido**: Usa una técnica llamada "delegación de eventos" para escuchar clics en los botones "Añadir al Pedido". Cuando se hace clic, añade el item a un array (una lista) llamado `pedidoActual`.
    - **Calcular Cuenta**: Después de añadir un item, recalcula el subtotal, el IGV (18%) y el total, y actualiza los valores en la sección "Mi Pedido".
    - **Finalizar y Pagar**: Al hacer clic en este botón, crea un objeto `factura` con el resumen de la compra y lo envía con un `fetch` de tipo `POST` a `server2.js` para que sea guardado.
    - **Limpiar Pedido**: Vacía el array `pedidoActual` y actualiza la interfaz.

---
3. ARCHIVOS DEL SERVIDOR (Lo que se ejecuta con Node.js)
---

Estos son tres servidores muy simples creados con el módulo `http` de Node.js. Su única función es leer y escribir en los archivos `.json` correspondientes.

*   `server1.js` y `server1.json`:
    - `server1.json`: Es una base de datos simple en formato de archivo. Contiene una lista (array) de objetos, donde cada objeto es un plato con "nombre" y "precio".
    - `server1.js`:
        - Si recibe una petición `GET` a `/platos`, lee el contenido de `server1.json` y lo devuelve.
        - Si recibe una petición `POST` a `/platos`, lee el `server1.json`, añade el nuevo plato que viene en la petición y vuelve a guardar el archivo actualizado.

*   `server2.js` y `server2.json`:
    - `server2.json`: Almacena un historial de todas las facturas generadas. Inicialmente está vacío.
    - `server2.js`:
        - Solo responde a peticiones `POST` a `/facturas`.
        - Cuando recibe una, añade los datos de la nueva factura al archivo `server2.json`.

*   `server3.js` y `server3.json`:
    - `server3.json`: Contiene la lista de tragos disponibles.
    - `server3.js`:
        - Es el más simple. Solo responde a peticiones `GET` a `/tragos`.
        - Cuando recibe una, lee el archivo `server3.json` y lo devuelve al cliente (`js/main.js`).
